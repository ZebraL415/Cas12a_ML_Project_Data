# 99_notes

本目录保存审计、证据、问题和决策记录。它用于解释数据为什么这样整理，而不是存放训练数据。

## 常用文件

- `*_data_audit*.md`：数据审计报告，说明读取了哪些文件和 sheet。
- `run_report*.md`：脚本运行报告，记录输入、输出、分类判断和质量问题。
- `evidence_trace*.md`：关键分类或标签判断的证据链。
- `problems_to_resolve*.md`：仍需人工确认的问题。
- `method_notes*.md`：可转写到论文 methods 的流程说明。
- `meeting_decisions.md`：会议或人工决策记录。

## 使用原则

- 不确定问题写在这里，不要在数据表里猜测。
- 每轮整理应留下可复现脚本和 run report。
- 中英文说明文件尽量保持一一对应，避免信息缺漏。

## 归档

历史备份放在 `_archive/backups/`。目录首页保留当前有效报告和问题清单。
