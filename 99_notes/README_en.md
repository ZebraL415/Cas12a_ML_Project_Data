# 99_notes

This directory stores audit reports, evidence traces, unresolved questions, and decisions. It explains why the data are organized in a given way. It does not store training data.

## Common files

- `*_data_audit*.md`: data audit reports describing scanned files and sheets.
- `run_report*.md`: script run reports recording inputs, outputs, classification decisions, and quality issues.
- `evidence_trace*.md`: evidence chains for key classification or label decisions.
- `problems_to_resolve*.md`: questions requiring manual confirmation.
- `method_notes*.md`: method-style workflow notes for future manuscripts.
- `meeting_decisions.md`: meeting notes or human decisions.

## Usage rules

- Record uncertain issues here instead of guessing inside data tables.
- Each round of data organization should leave a reproducible script and a run report.
- Chinese and English documentation should stay aligned to avoid missing or inconsistent information.

## Archive

Historical backups are stored in `_archive/backups/`. The directory front page keeps current active reports and question lists.
